Guía de configuración y despliegue para el equipo de desarrollo

Stack: React · Node.js · PostgreSQL · Render · Vercel

Parte 1 — Configurar el proyecto en una nueva computadora

Seguir estos pasos en orden. Solo se realizan una vez por máquina.

1 Instalar herramientas

  • Node.js (versión LTS) → https://nodejs.org → instalar con opciones por defecto
  • Git → https://git-scm.com → instalar con opciones por defecto
  • VS Code → https://code.visualstudio.com
Después de instalar Node.js, reiniciar VS Code para que reconozca el comando npm.

2 Clonar el proyecto desde GitHub

Abrir VS Code → menú Terminal → New Terminal → cambiar terminal a Git Bash:

# Ir a la carpeta de inicio y clonar
cd ~
git clone https://github.com/dianabejar-stack/jjseafoods.git
cd jjseafoods

3 Instalar dependencias

# Dependencias del servidor
cd server
npm install

# Dependencias del cliente
cd ../client
npm install

4 Crear el archivo .env del servidor

cd ../server

Crear un archivo llamado .env (sin extensión) con el siguiente contenido:

PORT=3001
DATABASE_URL=postgresql://jjseafoods_db_user:BTJq5u04yStKFZ5nJfKUYZIH8oiuthqg@dpg-d6mvr6s50q8c73as0avg-a.oregon-postgres.render.com/jjseafoods_db
Importante: Este archivo .env nunca se sube a GitHub (está protegido en .gitignore). Cada persona del equipo debe crearlo manualmente en su computadora.

5 Crear el archivo .env del cliente

cd ../client

Crear un archivo llamado .env con este contenido:

VITE_API_URL=
Dejar vacío para desarrollo local. El proxy de Vite redirige las llamadas /api al backend automáticamente.

6 Levantar el proyecto localmente

Abrir dos terminales en VS Code:

Terminal 1 — Backend

cd ~/jjseafoods/server
npm run dev

Debe mostrar:
Servidor corriendo en puerto 3001

Terminal 2 — Frontend

cd ~/jjseafoods/client
npm run dev

Debe mostrar:
Local: http://localhost:5173

Abrir http://localhost:5173 en el navegador para ver la aplicación.

Parte 2 — Desplegar cambios a producción

El despliegue es automático: basta con hacer git push a GitHub.

1 Flujo de despliegue

Editar código
en VS Code
→
git commit
+ git push
→
GitHub
detecta cambio
→
Render
redespliega API
+
Vercel
redespliega web

2 Comandos Git del día a día

# 1. Obtener los últimos cambios del equipo (hacer SIEMPRE antes de empezar)
git pull

# 2. Ver qué archivos cambiaron
git status

# 3. Agregar los archivos modificados
git add nombre-del-archivo.js
# (o para agregar todos los cambios)
git add .

# 4. Guardar cambios con mensaje descriptivo
git commit -m "descripción de lo que se cambió"

# 5. Subir a GitHub → dispara redespliegue automático
git push

Parte 4 — Solución a problemas comunes

❌ npm no se reconoce como comando

Reiniciar VS Code después de instalar Node.js. Si sigue fallando, verificar que Node.js quedó instalado ejecutando node --version en Git Bash.

❌ git push pide contraseña y falla

GitHub ya no acepta contraseñas normales. Se necesita un Personal Access Token:

  1. Ir a GitHub → avatar → Settings → Developer settings → Personal access tokens → Tokens (classic)
  2. Clic en Generate new token (classic)
  3. Marcar el permiso repo → generar
  4. Copiar el token y usarlo como contraseña cuando Git lo pida

❌ El dashboard muestra ceros en producción

El backend de Render en plan gratuito se "duerme" tras 15 min de inactividad. Al primer acceso tarda ~30 segundos en despertar. Esperar y recargar la página.

❌ Los cambios no aparecen en producción

  1. Verificar que el git push fue exitoso (sin errores en terminal)
  2. Revisar el panel de Render para ver el estado del despliegue del backend
  3. Revisar el panel de Vercel para ver el estado del despliegue del frontend

❌ Error al conectar a la base de datos localmente

Verificar que el archivo server/.env existe y contiene la variable DATABASE_URL correctamente (ver Paso 4).
